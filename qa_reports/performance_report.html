7 Recommendations
1
Fix Nginx Health Check Proxy — Add location /health { proxy_pass http://backend_servers; } to nginx.conf. This lets load balancers, Kubernetes probes, and uptime monitors verify backend health without bypassing Nginx.
2
Switch to Redis-Backed Rate Limiter — Replace the in-memory express-rate-limit with rate-limit-redis store. Key rate limits by JWT user ID or API key instead of IP to correctly handle multi-user load from the same origin.
3
Increase Nginx worker_connections to 4096–10000 — Compiler jobs hold HTTP connections open while waiting for job.finished(). At 500+ concurrent users, 1024 connections will be exhausted. Update nginx.conf: worker_connections 4096;
4
Add Bull Job Timeouts — Currently job.finished() can block indefinitely. Set a job timeout in Bull: enqueueCompilerJob(..., { timeout: 30000 }) and return a meaningful error to the client instead of a silent hang or 60s timeout.
5
Replace Blocking job.finished() with WebSocket / SSE — The current pattern blocks an Express server thread per compilation request. At scale, use WebSocket or Server-Sent Events to push results asynchronously, freeing API server threads immediately after job enqueue.
6
Increase Worker Concurrency — WORKER_CONCURRENCY=2 per worker = 24 total concurrent jobs. For a public compiler, consider increasing to 4–6 per worker (if host CPU allows), giving 48–72 parallel jobs and significantly reducing queue wait times under load.
